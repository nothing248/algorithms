# 🚴 外卖骑手算法进化史 —— 磁盘调度与电梯算法交互式演示平台

这是一个使用 **Streamlit** 构建的高颜值、可交互的算法科普演示 Demo，旨在生动形象地可视化计算机底层的磁盘调度算法与物理电梯调度算法。

本 Demo 是对公众号文章 **《电梯算法：从外卖骑手到看电梯算法的终极进化》** 的完美实测落地版。通过将复杂的“磁头寻道请求”映射为“外卖骑手派送订单”，让读者能够一边操纵骑手送外卖，一边秒懂底层硬核算法！

---

## 🌟 核心功能特色

1. **📖 骑手故事卡片（Tab 1）**
   - 完美呼应公众号文章人设。为 5 位骑手（先来先服务的小明、近路优先的小刚、撞墙掉头的老张、智慧老张、循环单行道老张）定制了极具现代科技感的毛玻璃故事介绍卡片，并清晰对照官方学术名词。
2. **📊 多算法擂台同台竞技（Tab 2）**
   - 在**同一组订单序列**和**骑手初始位置**下，并发测试所有 6 种算法（FCFS, SSTF, SCAN, LOOK, C-SCAN, C-LOOK）。
   - 输出包括总路程、送餐路程、边界空扫、闪现回扫、平均等待时间、最大等待时间（体现倒霉顾客的等待极限，暴露出 SSTF 的饥饿风险）以及波动差（体现公平性）等维度的性能矩阵和交互式 Plotly 对比图表。
3. **🚴 实时骑行平滑模拟器（Tab 3）**
   - 纯 HTML/CSS 动效渲染虚拟“美食一条街”（0-100号商铺），骑手 🚴 自行车图标会在街道上**平滑滑动送餐**。
   - 上方的累积路程表、右侧的历史轨迹图以及动态配送广播日志，全部随动画步骤**实时连贯更新**。
   - 支持在左侧控制台**自定义动画刷新速度（0.02秒 - 0.50秒）**，方便慢速观察或快速过图。

## 🛠️ 本地安装与运行方法

如果你希望在本地运行和开发该平台：

### 1. 克隆 / 下载本仓库代码
确保你已下载 `app.py` 以及依赖配置文件。

### 2. 安装依赖包
在控制台执行以下命令安装运行所需的第三方库（Streamlit, Pandas, Plotly, NumPy）：
```bash
pip install -r requirements.txt
```

### 3. 启动 Streamlit 服务
运行以下命令启动本地 Web 服务：
```bash
streamlit run app.py
```
终端会输出本地访问 URL（通常是 `http://localhost:8501`），自动在浏览器中打开演示界面。

## ☁️ 部署到 Streamlit Community Cloud

本 Demo 已配置好所有云端部署所需的依赖声明。你只需几步，即可免费将其发布到互联网：

### 第一步：将代码上传至 GitHub 仓库
1. 创建一个新的 GitHub 公开（Public）仓库；
2. 将 `app.py`、`requirements.txt` 和本 `README.md` 提交并 Push 到 GitHub 仓库中。

### 第二步：登录 Streamlit 云平台
1. 访问并登录 [Streamlit Community Cloud](https://share.streamlit.io/)；
2. 建议使用 GitHub 账号直接登录（以便自动关联仓库）。

### 第三步：新建部署（Deploy an app）
1. 点击右上角的 **"New app"** 按钮；
2. 在部署表单中进行配置：
   - **Repository**: 选择你刚刚上传的 GitHub 仓库名；
   - **Branch**: 填入你的主分支名称（通常是 `main` 或 `master`）；
   - **Main file path**: 填入入口 Python 文件 `app.py`；
3. 点击底部的 **"Deploy!"** 按钮。

系统将在 1~2 分钟内自动拉取依赖并构建完毕，生成一个公共的访问链接。届时，你就可以直接将该链接分享到你的公众号文章、朋友圈或技术群中，供读者在线操纵交互！

## 🌌 部署到魔搭社区 (ModelScope) 创空间

本 Demo 同样支持在阿里魔搭社区（ModelScope）创空间中部署。通过仓库根目录下的 `ms_deploy.json`，系统会自动将启动入口指向 `app.py`。

### 第一步：将代码上传至魔搭创空间 Git 仓库
1. 在魔搭社区中，点击 **"创空间"** $\rightarrow$ **"创建创空间"**；
2. 填写创空间信息，其中 **"SDK类型"** 选择 **"Streamlit"**；
3. 创建成功后，使用 Git 将本地代码（包括 `app.py`、`requirements.txt`、`ms_deploy.json` 及 `README.md`）推送（Push）到魔搭分配的远程 Git 仓库。

### 第二步：ModelScope 云端自动构建
1. 代码推送完成后，魔搭社区云端检测到 `ms_deploy.json` 和 `requirements.txt` 会自动安装依赖并构建部署；
2. 构建成功后，即可在创空间页面在线浏览并交互！
