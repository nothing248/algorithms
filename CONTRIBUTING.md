# 🤝 参与贡献指南 (Contributing Guide)

感谢你关注本项目！我们非常欢迎社区开发者为本项目贡献新的算法讲解与交互式演示。
本指南将帮助你了解如何新增一个算法模块，以及如何确保其符合项目的结构和美学规范。

---

## 📂 贡献流程

### 1. 新增算法目录规范

在根目录下创建你的算法文件夹，文件夹名称建议使用**英文小写加中划线**（例如：`binary-search-tree` 或 `dijkstra-pathfinding`）。

每个算法目录**必须**包含以下文件：
*   **`README.md`**：详细的算法原理讲解、数学模型描述，以及运行说明（如适用）。

**可选（但强烈推荐）**：
*   **交互式主程序**（非必须，但有的话更好）：
    *   若是 Python 项目，推荐使用 Streamlit，入口文件命名为 `app.py`。
    *   若是 Web 项目，推荐使用 Vanilla JS 或现代前端框架，入口文件命名为 `index.html` 或对应前端规范的运行命令。
*   **依赖声明**（当提供了主程序且需要依赖时）：
    *   Python 项目提供完整的 `requirements.txt`。
    *   NodeJS 项目提供 `package.json`。

### 2. 文档编写规范

每个算法目录下的 `README.md` 应遵循以下内容框架：
1.  **算法简介**：用通俗易懂的语言解释该算法要解决什么问题。
2.  **核心原理**：图文并茂或使用 LaTeX 公式详细推导/解释算法逻辑。
3.  **应用场景**：该算法在现实世界中的实际用途。
4.  **交互说明**（若提供交互程序）：介绍在交互式界面中可以进行哪些操作，观察到哪些现象。
5.  **运行方式**（若提供交互程序）：列出本地启动演示程序的具体命令。

### 3. 交互式界面设计美学规范（若提供交互界面）

为了维持项目整体的高品质视觉体验，如果您提供了交互页面，设计时请遵循以下原则：
*   **优选精致配色**：避免使用浏览器默认的纯红、纯蓝、纯绿。使用精心调配的 HSL 配色方案或深色主题，确保色彩的和谐与高对比度。
*   **顺滑的微动画**：在状态转移（如排序交换元素、图节点寻路遍历）时，提供平缓的过渡动画，避免生硬的画面闪烁。
*   **清晰的参数控制**：控制面板应布局合理，对于复杂参数（如速度、规模、策略类型）提供滑块（Slider）、下拉菜单（Select）等直观的控制组件。
*   **实时反馈**：算法每次迭代或每步执行时，界面应对当前状态（如当前搜索指针位置、已确定的最短路径）进行高亮提示。

## 🛠️ 提交 PR (Pull Request) 的步骤

1.  **Fork** 本仓库到你自己的 GitHub 账号下。
2.  在本地克隆你 Fork 的仓库，并创建新分支：
    ```bash
    git checkout -b feature/add-[algorithm-name]
    ```
3.  按照规范在本地完成算法代码及文档的编写，并进行充分测试。
4.  提交你的修改并推送到远程仓库：
    ```bash
    git add .
    git commit -m "feat: add [algorithm-name] visualization and docs"
    git push origin feature/add-[algorithm-name]
    ```
5.  在 GitHub 上发起 Pull Request，并详细描述你的实现和运行截图。我们会在第一时间进行 Review！
